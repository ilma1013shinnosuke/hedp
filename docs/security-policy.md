# 秘密情報と実データの方針

## 分類

認証情報、トークン、Cookie、暗号鍵は秘密情報として扱う。実機ID、家庭内IP、
外部サービスの家庭固有IDは認証情報でなくても環境固有情報として扱う。生活履歴、
センサー値、電力値、画像、操作履歴は個人データとして扱う。

## 保存場所

- SwitchBotの実行用認証情報は、Git管理外・権限0600の `.env` に置く。
- FusionSolarの現行launchd実行値は、リポジトリ外・権限0600のplistにある。
- 引き継ぎ文書には秘密値を書かず、保存場所と環境変数名だけを書く。
- Gitには値のない `.env.example`だけを置く。
- 将来は暗号化された秘密情報保管庫を原本とし、実行時の値を展開する。

## 禁止事項

- 秘密値を回答、ログ、スクリーンショット、テスト、文書、Gitへ出さない。
- 実データ、DB、バックアップ、ログをGitへ追加しない。
- 実際の認証値を模擬データとして再利用しない。
- 古いトークンやCookieをアーカイブしない。
- 稼働中SQLiteをクラウド同期フォルダへ直接置かない。

## 権限

`.env`、引き継ぎ文書、実DB、バックアップ、runtimeファイル、運用ログ、生成plistは
原則として所有者だけが読み書きできる0600とする。ディレクトリは必要最小限の
権限にする。

## Git履歴

秘密値らしい項目名だけでなく、現在の実行値そのものを値を表示せず照合する。
秘密値が履歴に入った場合は、ファイル削除だけでは解決しない。値を失効・再発行し、
必要に応じて履歴を書き換える。履歴書換えは共有先への影響を確認して別作業で行う。

## 現在の例外

検証済みのFusionSolar機器識別子の一部は、既存コード、テスト、知識文書に残っている。
これは認証パスワードではないが家庭固有情報である。環境変数へ移すと現在のlaunchd
設定と既定動作が変わるため、メーカー別コードの移行時に、テスト用の架空IDと
実環境値へ分離する。切替前にinstallerの再実行対象を確認する。
