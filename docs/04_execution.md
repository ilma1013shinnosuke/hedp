# ④ 操作・実行

## 目的

③が作ったメーカー非依存の操作意図を、Smart LEDZ、Qrio、SwitchBotなどの具体的な
命令へ変換し、安全に実行して結果を確認する。

## 実行前

対象機器、部屋、現在状態、利用者権限、安全条件、操作期限を確認する。鍵、火気、空調
など影響の大きな操作は、再認証、明示確認、在宅状態など追加条件を設ける。

## 実行中と実行後

- 同じ依頼を重複実行しない識別子を持つ。
- タイムアウト、再試行回数、総実行時間に上限を設ける。
- 要求を送ったことと、機器状態が実際に変わったことを区別する。
- 成功、失敗、不明、見送りを記録する。
- 操作意図、判断理由、対象、実行者、結果を追跡できるようにする。

## 故障時

④が停止しても①〜③と保存データを壊さない。HEDPが使えなくても、メーカー純正アプリ
や手動操作を残す。機器自身の安全機能をHEDPで置き換えない。

## 現在の状態

調査用の個別操作はできるが、共通の安全な④はまだ実装していない。そのため
`src/hedp/execution/`は作らない。最初の正式な操作機能を実装するときに、対象確認、
重複防止、結果確認、監査記録を同時に設計する。
