# ② 情報蓄積

## 目的

①で得た事実を失わず、後から再現・確認・分析できる形で保存する。大まかには
「証拠を残す」と「使える形で残す」の両方を担当する。

## 保存する種類

- RawData: メーカーの完全応答と取得条件を、変更しない証拠として保存する。
- Record: RawDataから再現できる、項目名・値・単位をそろえたデータを保存する。
- 現在値: 最後に確認できた状態と、その取得時刻・鮮度を示す。
- 時系列履歴: 値や状態の変化を時刻順に保持する。
- イベント: センサー反応、アラーム、操作、失敗など一回ごとの出来事を保持する。

## 原則

- 欠損を0で埋めない。推測値を観測事実として保存しない。
- RawDataを通常の処理で削除、上書き、変更しない。
- 正規化規則を直した場合、RawDataから再構築できるようにする。
- 同一事実の重複は防ぐが、異なる取得時刻の現在値は時系列証拠として保持する。
- 値だけでなく取得元、品質、鮮度、変換方法を追跡できるようにする。

## 現在の実装

共通のRawData、Record、SQLite保存は `src/hedp/storage/` にある。SwitchBotは専用の
機器台帳、観測、取込監査、競合、欠損、集計テーブルも持つ。統合のためだけに既存DB
構造を変更せず、実際の利用要件が明確になった時点で共通化を判断する。

## 容量

現行RawDataは `hedp.db` 内にある。整理作業では複製も移動もしない。保存期間、圧縮、
別ストレージ化は、復元可能性と必要な履歴を確認する独立作業として行う。
