# 作業領域監査（2026-07-22）

## 結論

SumiCoreのコード領域は小さく保たれており、GitへDB、Raw、バックアップ、ログ、画像、
生成plist、家庭固有設定、退避仮想環境は入っていない。追跡ファイルは119件で、最大の
ファイルも約31 KBだった。現時点で容量整理を目的としたRawの複製や移動は不要である。

`.env.example`は秘密値を持たない項目名のテンプレートとして意図的に追跡する。
認証に関する単語を含むコードと文書は存在するが、これは設定の読み込み、署名、検査、
テスト、運用説明に必要なものである。実値との照合は値を表示しない専用検査で扱う。

## 生成物とアーカイブ候補

次はすべてGit管理外の再生成可能ファイルで、合計も小さいため今は削除しない。

- `.pytest_cache/`
- `.ruff_cache/`
- `build/`
- `src/sumicore.egg-info/`
- Pythonの`__pycache__/`

現役コードと過去資料が混ざるような大きなアーカイブ候補は見つからなかった。
GASを廃止する場合だけ、方針どおりリポジトリ外の
`hedp-archive/cloud/gas/<vendor>/`へ移し、秘密値は移さない。

## 旧名称

`hedp`と`HEDP`は、Python import、旧CLI、環境変数fallback、DB・ログ・lockの現役名として
残っている。これは段階移行の互換性を保つためであり、直ちに一括置換しない。
表示名、配布名、新CLI、launchd labelはSumiCoreへ移行済みである。

DB・ログ・lock、GitHubリポジトリ、ローカルパスの改名は、24時間監視の合格後も別工程に
分ける。稼働DBをコピーせず、収集停止中の同一ディスク移動を原則とする。

## Windows互換性

Python本体は3.11以上と一般的な依存パッケージだけで構成され、基本機能はWindowsへ
移植可能である。一方、launchd installer、plist、`launchctl`、macOSの絶対パスはMac専用の
運用層である。Windows移行時はPythonコードを共用し、次だけを別実装する。

- タスクスケジューラへの登録・解除
- `.env`と家庭固有設定のアクセス権設定
- ログ、DB、lock、バックアップの標準保存場所
- タイムアウト、単一起動、失敗時復旧のPowerShellラッパー

Mac用スクリプトを無理に共通化せず、OS別運用アダプターとして分ける。

## GAS版FusionSolarの残作業

ローカルコード、重複抑止、応答上限、認証切れ検知、Driveキュー、Mac側の検査・取込は
実装済みである。残っているのは外部環境を変更する次の作業である。

- Apps Scriptプロジェクトと専用Driveフォルダの作成
- Script Propertiesへの実値設定
- OAuth権限の確認
- 手動取得、重複判定、認証切れ通知の実環境検証
- 日次triggerの登録
- ダウンロードした実データのdry-runと、別承認後の初回DB取込

自動再認証は未実装である。現在はCookieとCSRFの期限切れを検知して停止・通知する設計で、
ユーザー名やパスワードをGASへ追加しない。

## 次の優先順位

1. SumiCore本番切替の24時間監視を完了する。
2. 本番`main`と監視記録をGitHubへ安全に反映する。
3. GAS配備の対象プロジェクト、Drive、実APIへの影響を確認して承認を得る。
4. GASを実環境へ配備し、最初はDBを変更しない検査だけ行う。
5. 改名とWindows対応は現行収集の安定後に独立した計画として進める。
