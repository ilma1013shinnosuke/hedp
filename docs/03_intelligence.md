# ③ 情報利用・判断

## 目的

蓄積した事実とリアルタイムの出来事から、可視化、分析、予測、自動化の判断を行う。
メーカーごとの命令ではなく、「浴室を点灯する」など家庭内で意味のある操作意図を作る。

## 区別するもの

- 事実: 実際に取得・保存された値や出来事。
- 推定: 在室、入浴、故障兆候など、複数の事実から導いた可能性。
- 判断: 推定や条件を基に、何をするべきか決めた結果。

推定と判断には、使用した事実、規則やモデルの版、確信度、判断時刻を残す。後から
「なぜそう判断したか」を説明できない自動化は行わない。

## 原則

- 欠損を正常値と解釈しない。
- 一つのセンサーだけで安全性の高い判断を確定しない。
- メーカー固有API、トークン、機器コマンドを知らない。
- いきなり自動操作せず、可視化、提案、影運転、半自動、自動の順に検証する。
- 判断がなくても①収集と②蓄積は継続できる。
